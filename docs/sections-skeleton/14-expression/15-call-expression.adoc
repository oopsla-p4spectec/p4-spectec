[#sec-call-expression]
=== Call expressions

// Added
Call expressions are used to invoke callables or constructors.

// Added
${syntax: callExpression
          callTarget
          callableTarget
          constructorTarget}

// Added
Details of how calls are resolved and evaluated are described in <<sec-call>>.

==== Type checking

// Added
After type checking, call expressions are represented as:

// Added
${syntax: callExpressionIR callableTargetIR constructorTargetIR}

[discrete]
===== Calling a callable

// Added
${rulegroup-source: Expr_ok/callExpression-callable}
${rulegroup-prose: Expr_ok/callExpression-callable}

[discrete]
===== Calling a constructor

// Added
${rulegroup-source: Expr_ok/callExpression-constructor}
${rulegroup-prose: Expr_ok/callExpression-constructor}

==== Local compile-time evaluation

// Added
Certain call expressions can be evaluated during type checking:

// Added
${rulegroup-source: Expr_eval_lctk/callExpressionIR}
${rulegroup-prose: Expr_eval_lctk/callExpressionIR}

==== Compile-time evaluation

// Added
Call expressions may also be evaluated at compile time when instantiation phase
takes place. A constructor call evaluates to an instance of the constructed
type, while a callable call evaluates to the result of the call.
