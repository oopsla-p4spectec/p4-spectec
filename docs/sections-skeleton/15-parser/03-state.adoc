=== Parser states

// 763
A parser state is declared with the following syntax:

// 764 - Modified
${syntax: parserState}

// 765
Each state has a name and a body. The body consists of a sequence of statements
that describe the processing performed when the parser transitions to that
state, including:

// 766
- Local variable declarations,
- Assignment statements,
- Method calls, which serve several purposes:
  * Invoking functions (e.g., using `verify` to check the validity of data already parsed), and
  * Invoking methods (e.g., extracting data out of packets or computing
    checksums) and other parsers (see <<sec-sub-parser>>), and
- Conditional statements,
- Transitions to other states (discussed in <<sec-transition-statement>>).

// 769
Architectures may place restrictions on the expressions and statements that can
be used in a parser--e.g., they may forbid the use of operations such as
multiplication or place restrictions on the number of local variables that may
be used.
